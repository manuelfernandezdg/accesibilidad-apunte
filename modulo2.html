    Imágenes a través de CSS

    Una práctica recomendada en accesibilidad es que si la imagen que utilizamos para la interfaz es "decorativa" o no agrega información adicional al contenido principal, la misma debe ser llamada desde CSS. Esto evita tener que agregar información adicional a través del atributo alt dentro de HTML

    Si trabajamos con íconos en botones/links.

    Este ejemplo está "bien" en principio, manejamos el propósito del link con el atributo aria-label y no es necesario agregar texto adicional dentro del botón.

    1<button class="btn-fav" aria-label="Agregar a favoritos">
    2  <!-- background-image: url(icon-fav.svg); en CSS -->
    3</button>

    En este segundo ejemplo tenemos una solución mucho más robusta. Si los usuarios utilizan un traductor automático, el valor del aria-label se pierde y no es traducido. En cambio, el texto dentro del .sr-only es parte del contenido del DOM, es traducible, seleccionable y visible si falla el CSS.

    1<button class="btn-fav">
    2  <span class="sr-only">Agregar a favoritos</span>
    3  <!-- el ícono puede ir como background del button o como SVG inline -->
    4</button>

    Otra forma puede ser utilizar la imagen de icono dentro del botón o link, junto con la etiqueta de descripción del propósito de la acción. En este caso como la etiqueta es visible no es necesario utilizar sr-only para ocultar el texto. Utilizamos aria-hidden = "true" en la imagen para que el lector de pantalla ignore el elemento y no sea redundante.

    1<button class="btn-fav">
    2  <svg aria-hidden="true" ...></svg>
    3  <span>Favoritos</span>
    4</button>

    También podemos utilizar la imagen del ícono directamente dentro del botón o link, donde el alt sirve al propósito de explicar la función del botón, sin la necesidad de uso de aria o .sr-only.

    1<button>
    2  <img src="fav.svg" alt="Agregar a favoritos">
    3</button>

    ¿Qué es .sr-only?

    El patrón de diseño sr-only es una clase que utilizamos en accesibilidad para ocultar un elemento de la vista base de nuestro diseño y es accesible solamente desde lectores de pantalla. De ahí su nombre en inglés screen reader only.

    1.sr-only {
    2  position: absolute;
    3  width: 1px;
    4  height: 1px;
    5  padding: 0;
    6  margin: -1px;
    7  overflow: hidden;
    8  clip: rect(0, 0, 0, 0);
    9  white-space: nowrap;
    10  border: 0;
    11}

    Esta clase quita el elemento del flujo normal del HTML a través de la propiedad position: absolute. Lo reduce a la expresión mínima visible y oculta su contenido a través de las propiedades width/heigh y clip. También evita que el texto se expanda y sea visible con la propiedad white-space: nowrap

    Es importante no utilizar display: none, visibility: hidden o hidden porque estas propiedades ocultan el elemento del árbol de accesibilidad.

    WAI-ARIA

    Las ARIA (Accesible Rich Internet Applications) son un conjunto de atributos de HTML que sirven para suplir las limitaciones de accesibilidad cuando el HTML semántico no alcanza.

    • aria-label="texto" Etiqueta invisible que reemplaza el texto visible.
    • aria-labelledby="id" Referencia a otro elemento que sirve como etiqueta.
    • aria-describedby="id" Referencia a otro elemento con descripción extendida.
    • aria-controls="id" Indica que este elemento controla otro.
    • aria-current="page" Elemento activo en un conjunto de navegación.
    • aria-hidden="true" Oculta un elemento del árbol de accesibilidad.

    Estados aria

    Podemos utilizar atributos aria no sólo para identificar qué tipo de elemento describimos para la API de accesibilidad sino también para describir su estado.

    • aria-expanded="true/false" Acordeón, menú desplegable — ¿está abierto?
    • aria-pressed="true/false" Botón de tipo toggle
    • aria-selected="true/false" Pestaña activa en un tabpanel
    • aria-disabled="true" Elemento deshabilitado (semántico, no visual)
    • aria-invalid="true" Campo de formulario con error de validación
    • aria-required="true" Campo obligatorio

    Lectura del nombre accesible

    El accesible name es cómo los lectores de pantalla identifican un elemento. Esta es la prioridad, de mayor a menor:

    • aria-labelledby (referencia a otro elemento)
    • aria-label (texto directo)
    • Texto visible del elemento (contenido textual interno)
    • title (último recurso, mala accesibilidad)

    aria-label reemplaza al texto visible. Si el a dice "Favoritos" y le colocamos aria-label="Acceder a favoritos", el lector de pantalla lee "Acceder a favoritos", no "Favoritos". Es recomendable usarlo sólo cuando no haya texto visible o necesitemos un texto alternativo diferente.

    Roles role=""

    El atributo role define qué es un elemento para la API de accesibilidad. El uso de role quedó anticuado gracias a que es preferible utilizar html semántico.

    • role="button" <button>
    • role="navigation" <nav>
    • role="banner" <header>
    • role="bontentinfo" <footer>
    • role="main" <main>
    • role="heading" <h1 / h6>

    Los roles ayudan a navegar la página por regiones. Los elementos HTML semánticos ya exponen su rol implícito, no es necesario utilizar role.

    En el caso de que haya más de un header en la página, podríamos señalar el principal con el role="banner"

    Si tenemos más de un nav, podemos distinguir utilizando aria-label para mostrar su función. Por ejemplo: aria-label="Menú principal", aria-label="Redes sociales"

    Patrones de navegación / interacciones comunes con aria

    Skip link (salto de navegación)

    1<a href="#main-content" class="sr-only">Saltar al contenido principal</a>
    2<main id="main-content">

    Acordeón

    1<button aria-expanded="false" aria-controls="panel-1">Pregunta 1</button>
    2<div id="panel-1" role="region" aria-labelledby="...">
    3  Respuesta 1
    4</div>

    Pestañas / Tabs

    1<div role="tablist" aria-label="Secciones del curso">
    2  <button role="tab" aria-selected="true" aria-controls="panel-1" id="tab-1">HTML</button>
    3  <button role="tab" aria-selected="false" aria-controls="panel-2" id="tab-2">CSS</button>
    4</div>
    5<div role="tabpanel" id="panel-1" aria-labelledby="tab-1">...</div>
    6<div role="tabpanel" id="panel-2" aria-labelledby="tab-2">...</div>
