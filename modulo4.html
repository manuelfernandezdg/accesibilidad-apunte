    Robusto

    El principio robusto establece que el contenido debe ser lo suficientemente robusto como para que pueda ser interpretado de forma fiable por una amplia variedad de agentes de usuario, incluidas las tecnologías de asistencia.

    Este principio asegura compatibilidad, también que sea a prueba de futuros cambios en navegadores, lectores de pantalla o tecnologías de asistencia.

    El enfoque principal es un correcto uso de tecnologías web, como el HTML semántico, ARIA bien implementado, código válido.

    Formulario robusto:

    1<form id="mi-formulario" novalidate>
    2    <div class="campo">
    3        <label for="email">Correo electrónico <span aria-hidden="true">*</span></label>
    4        <input

    14    
    15    <button type="submit">Enviar formulario</button>

    2<nav aria-label="Menú principal">
    3    <!-- enlaces -->
    4</nav>
    5 
    6<!-- Evitar redundancia -->
    7<button aria-label="Cerrar">✕</button>   <!-- Solo si el texto visible no es suficiente -->

    Checklist sobre principios operables:

    • Usar siempre HTML semántico
    • Validar el HTML (W3C Validator)
    • Todos los componentes interactivos tienen nombre accesible (visible o aria-label / aria-labelledby)
    • Roles ARIA correctos cuando se crean componentes personalizados
    • Estados actualizados (aria-expanded, aria-checked, aria-selected, etc.)
    • Mensajes de estado usan role="status" o role="alert" + aria-live
    • Soporte para prefers-reduced-motion, prefers-color-scheme, etc.
